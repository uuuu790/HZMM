# Design Context: HZMM — HumanitZ Mod Manager

## Target Audience
PC gamers who play HumanitZ (survival game) and use mods. Mostly intermediate-to-advanced users comfortable with modding tools. They use this app frequently during gaming sessions, often switching between it and the game itself.

## Use Cases
- Installing, toggling, and uninstalling game mods (.pak files and UE4SS Lua mods)
- Managing mod load order and resolving conflicts
- Creating backups before experimenting with new mods
- Quickly searching and filtering large mod lists
- Configuring individual mod settings
- Installing mods directly from URLs or Nexus Mods

## Brand Personality & Tone
**Modern, polished, gamer-adjacent desktop app** — not flashy esports aesthetics, but a refined tool that feels premium and trustworthy. Think "macOS app built for gamers" rather than "gaming HUD".

- **Calm and confident** — the UI should feel stable and in control, never chaotic
- **Tactile and responsive** — interactions should feel immediate with satisfying micro-animations
- **Dark-mode first** — most gaming sessions happen in low-light environments
- **Professional but approachable** — not cold enterprise software, but also not neon-drenched gaming aesthetic

## Design System

### Visual Language
- **Glassmorphism** — frosted glass cards (`bg-white/60 dark:bg-slate-900/60 backdrop-blur-xl`)
- **Extreme rounding** — pill buttons (`rounded-full`), large cards (`rounded-[2.5rem]`)
- **Dynamic accent color** — user-selectable theme color via CSS variables (`var(--accent-rgb)`, `var(--accent-500)`)
- **Slate base** — neutral slate color family for backgrounds and text

### Motion Principles
- Slide-up entrance animations with staggered delays on list items
- Hover lifts: `-translate-y-0.5` for interactive elements
- Subtle, purposeful — never decorative animation for its own sake

### Typography
- Clear hierarchy: bold headings, readable body text
- Functional over expressive — legibility in dense mod lists is critical

### Spacing
- Generous padding inside cards
- Tight list density for mod rows (users may have 50+ mods)
- Clear separation between sections

### Icons
- lucide-react icon set throughout
- Icon backgrounds: `rgba(var(--accent-rgb), 0.15)` — tinted, not solid

### What to Avoid
- Neon/cyberpunk aesthetics — too cliché for a mod manager
- Heavy gradients — the glassmorphism already provides depth
- Flat/material design patterns — inconsistent with the established glass aesthetic
- Harsh contrasts — the app should feel soft even in dark mode
